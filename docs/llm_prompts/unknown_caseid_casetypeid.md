Prompt: Why does Time Molecules need an Unknown CaseType and a special Unknown CaseID such as -1 for rare ad hoc events that do not belong to any ordinary case?

Abstract: Time Molecules benefits from having both an Unknown CaseType (dbo.CaseTypes.CaseTypeID) and a reserved Unknown CaseID (dbo.Cases.CaseID) so that rare, uncategorized, or one-off events can still be ingested without breaking the event model. The Unknown CaseType provides a formal classification for events whose case context is not yet known or does not fit a normal recurring process. The special Unknown CaseID, such as -1, acts as a durable placeholder case for exceptional events that are too infrequent or too unusual to justify creating a dedicated case instance at ingestion time. This supports liberal event capture while preserving referential consistency. It is especially useful for ad hoc events such as major disclosures, abrupt societal disruptions, or unusual external events whose analytical value is real even when they do not naturally belong to an established case structure.
