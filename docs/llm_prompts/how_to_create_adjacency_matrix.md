Prompt: How do the `dbo.sp_AdjacencyMatrix` stored procedure and legacy `dbo.AdjacencyMatrix` table-valued function create or display an adjacency matrix in TimeSolution, including their parameters, outputs, and recommended usage?

Abstract: An adjacency matrix in TimeSolution is a first-order transition summary over a selected event set, returned as rows rather than a wide pivot. Each result row represents an `EventA -> EventB` transition and includes `EventA`, `EventB`, `probability` as `P(B|A)`, `Event1A_Rows` as the total outgoing volume from `EventA`, and `count` as the raw transition count. The preferred implementation is `dbo.sp_AdjacencyMatrix`, which accepts `@EventSet`, `@enumerate_multiple_events`, optional `@transforms`, and optional `@SessionID`, and uses the `MarkovProcess2` work-table path for newer agent-oriented workflows. The legacy alternative is `dbo.AdjacencyMatrix(@EventSet, @enumerate_multiple_events, @transforms)`, a deprecated TVF that still returns the same transition summary by grouping `MarkovProcess` results and computing `count / Event1A_Rows`. The material also provides concrete execution examples using event sets such as `restaurantguest` and `poker`.
