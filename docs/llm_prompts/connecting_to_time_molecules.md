Prompt: How do I connect an AI agent or application to TimeSolution securely using approved SQL interface objects like views, stored procedures, table-valued functions, and scalar functions instead of direct table access? ([GitHub][1])

Abstract: TimeSolution is designed to be accessed through a controlled SQL interface rather than through direct reads of base tables. A valid connection requires a reachable server or endpoint, database name, appropriate driver or connector, authentication method, credentials or identity, network permission, and object-level database permissions. The intended security model grants access to approved views, stored procedures, TVFs, and scalar functions while keeping base tables locked down. Row-level visibility may further restrict results through mechanisms such as `AccessBitmap`. SQL Server is the active reference platform, Azure SQL Managed Instance is the closest planned deployment target, and Azure Synapse and Snowflake are planned targets that will require connector setup, security configuration, and some SQL adaptation before full support is available. ([GitHub][1])
