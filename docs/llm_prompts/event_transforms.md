Prompt: How do I map multiple raw event codes to broader event categories in TimeSolution before building sequences or Markov models?

Abstract: This material explains the concrete transform mechanism in TimeSolution for renaming event codes before modeling. A transform is a JSON object that maps original event names to replacement event names, such as collapsing multiple traffic events into a single `traffic` category. The file describes how `dbo.Transforms` stores reusable transform definitions with `TransformsKey`, `Code`, and `Description`; how `dbo.TransformsKey` computes the canonical key; how `dbo.InsertTransforms` inserts or reuses a transform definition; how `dbo.UpdateTransform` maintains stored transform metadata; and how `dbo.ParseTransforms` converts either raw JSON or a stored transform code into a normalized rowset of source-to-target mappings. It also explains that procedures such as `sp_Sequences` accept `@transforms` as part of the model definition, so two models that differ only by transform mapping are treated as different model configurations.
