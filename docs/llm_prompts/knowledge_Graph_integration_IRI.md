Prompt: How do I use and populate `IRI` columns in TimeSolution so lookup tables like `CaseTypes`, `DimEvents`, `DimAnomalyCategories`, and `SourceColumns` can link to a knowledge graph? 

Abstract: TimeSolution uses `IRI` columns in key lookup-style tables to give relational members a stable semantic identity that can connect outward to a knowledge graph. The material explains that `IRI` is used in tables such as `CaseTypes`, `DimEvents`, `DimAnomalyCategories`, and `SourceColumns`, where it serves as a durable identifier for the meaning of a case type, event, anomaly category, or source field. It also explains that `SourceColumns.Description` should help determine the correct RDF or ontology concept before assigning the `IRI`, and that `dbo.get_semantic_web_llm_values` already exports rows with `Description` and `IRI` for semantic-web and LLM-oriented downstream use. The file emphasizes careful setup: author or improve descriptions first, prefer stable ontology URIs, avoid weak or temporary links, and treat `IRI` as a governed semantic bridge rather than decorative metadata.
