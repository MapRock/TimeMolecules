Prompt: How can event properties in Time Molecules capture not just what happened, but also goals, forecasts, accumulated context, and the logic that caused an event to occur?

Abstract: Event properties in Time Molecules turn an event into more than a timestamped action by attaching structured JSON key-value pairs that describe different aspects of meaning around the event. ActualProperties record what was true when the event happened. IntendedProperties record the target or desired outcome. ExpectedProperties record the forecasted or predicted outcome, which may differ from the target. AggregationProperties add rolled-up context such as running totals, averages, or windowed measures across related events. An optional TriggerFunction records the rule, function, API call, or other logic that caused the event to fire, along with its input parameters. Together, these property types let an event carry factual state, intent, prediction, broader process context, and causal logic, making event streams more useful for auditing, reasoning, diagnostics, process improvement, and AI-driven analysis. :contentReference[oaicite:0]{index=0}
