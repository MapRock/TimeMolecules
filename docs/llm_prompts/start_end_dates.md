Prompt: How do I choose and generate the right `StartDateTime` and `EndDateTime` values in TimeSolution so I can build or compare models over meaningful time windows?

Abstract: This material explains how to use `dbo.TimeIntelligenceWindow(@AsOfDateTime, @Units, @FuncCode)` to generate consistent time windows before running TimeSolution queries and models. The function returns one row with `WindowStart`, `WindowEnd`, `WindowLabel`, and `Grain`, and supports current-period codes, to-date codes, previous and next full periods, lag and lead periods, rolling windows, and multi-period-to-date windows. The file emphasizes that date range selection is a core part of Time Molecules: many major functions depend on `@StartDateTime` and `@EndDateTime`, broad all-history ranges are usually analytically weak and expensive, and many of the best comparisons hold all other parameters constant while changing only time. The skill also explains the inclusive `WindowStart` and exclusive `WindowEnd` pattern and shows how the returned window can be fed into event queries and model-building workflows.
