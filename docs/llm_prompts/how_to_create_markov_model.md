* Prompt: How do you use the CreateUpdateMarkovProcess stored procedure to create or update a persistent Markov model as a first-class object in the TimeMolecules database?

* Abstract: In TimeMolecules, CreateUpdateMarkovProcess is the stored procedure that turns a Markov model into a reusable, first-class database object. It accepts an output @ModelID (which it creates or reuses), an @EventSet (as a CSV list of events or a resolving code), date-range parameters, @ByCase grouping, an optional @metric (defaulting to “Time Between”), @enumerate_multiple_events handling, optional JSON filters at case and event level, and @transforms for event-name normalization. The procedure first resolves the model definition through InsertModel (storing metadata and a ParamHash in the Models table), clears any prior transition rows for that model, calls MarkovProcess2 to compute fresh transition probabilities and statistics, and writes the results into ModelEvents as first-order segments such as EventA → EventB with associated metrics. When @InsertSequences is true, it also stores richer path detail in ModelSequences. Models with identical effective parameters resolve to the same ModelID, enabling refresh without duplication while populating the supporting tables for later search, comparison, embedding, or reuse.
