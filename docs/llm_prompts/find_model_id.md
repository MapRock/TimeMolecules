Prompt: How do I find the right model in TimeSolution so I can get its `ModelID` and use it in later steps?

Abstract: This material explains how to locate an existing `ModelID` in the TimeSolution database by using three concrete SQL objects: `dbo.ModelID` for exact lookup, `dbo.ModelsByParameters` for broader candidate searches, and `dbo.ModelsWithProperties` for browsing models by slicing properties. It defines the purpose of each object, lists the major inputs such as event set, enumeration behavior, date range, transforms, `ByCase`, metric, case and event property filters, model type, and access bitmap, and clarifies when to use exact matching versus broader search. It also shows that model visibility can be restricted by access filtering, that property filters are passed as flat JSON, and that a practical agent workflow is to try exact lookup first, then search for candidates, then continue into downstream objects like `dbo.ModelMatrix` after a `ModelID` is found.
