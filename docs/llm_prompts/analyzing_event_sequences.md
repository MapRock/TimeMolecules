Prompt: How can I analyze recurring event sequences across cases to see what tends to happen next, how common those paths are, and how stable they are over time?

Abstract: Sequence analysis in TimeSolution examines ordered event fragments that recur across many cases without building a full Markov model. It identifies specific event sequences, measures how often they occur, and shows which next events tend to follow them. The results include occurrence counts, conditional probabilities, distinct case counts, and timing statistics for both the full fragment and the final hop. The analysis is constrained by event sets, date ranges, transforms, case filters, event filters, metric choice, and access bitmap security, so the sequence results reflect the relevant slice of visible events rather than the entire event warehouse. This makes sequence analysis useful for studying candidate pathways, validating event sets, exploring what usually follows a given fragment, and understanding whether a recurring sequence is common, stable, or noisy.
