Prompt: How do I populate `STAGE.ImportEvents` and use `dbo.ImportEventsFromStage` to load staged events into Time Molecules, including `SourceID`, natural `CaseID`, JSON property columns, `AccessBitmap`, `CaseType`, and `NaturalKey_SourceColumnID`?

Abstract: `STAGE.ImportEvents` is a staging table for one event occurrence per row, with columns for source identity, natural case key, event name and description, event time, case-level JSON, several event-level JSON property buckets, load timestamp, access bitmap, case type, and optional source-column lineage. `dbo.ImportEventsFromStage` validates the staged rows, inserts new event types into `dbo.DimEvents`, maps staged natural case keys to new internal numeric case IDs, loads `dbo.Cases`, `dbo.CaseProperties`, `dbo.EventsFact`, and `dbo.EventProperties`, then refreshes parsed case and event properties for downstream filtering, drill-through, and Markov model generation. The tutorial also explains how to choose `SourceID` and `NaturalKey_SourceColumnID`, how to use `CaseProperties` and event property JSON as the practical equivalents of case and event filter properties, and how to monitor progress through the import logging path.
